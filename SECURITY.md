# 安全策略

## 支持范围

安全修复优先覆盖 `main` 分支和最新 Release。历史版本根据影响范围提供升级建议。

## 报告项目安全漏洞

请通过 GitHub 的 [Private Vulnerability Reporting](https://github.com/DHKun/dddd/security/advisories/new) 提交项目自身的安全漏洞。

报告内容建议包含：

- 受影响版本或 commit
- 漏洞影响和攻击前提
- 最小复现步骤或测试代码
- 建议修复方向

请勿在公开 Issue 中披露可直接利用的细节、真实目标、凭据或 Token。维护者会在确认报告后协调修复、公告和发布时间。
